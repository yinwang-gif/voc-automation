# Builder（最小验证方案）

Define the smallest verifiable product path that can expose whether the idea is real.

## Do

- Define user scenario and minimal closed loop.
- Split MVP / V1 / Later.
- Identify what can be fake, manual, concierge, or prototype-only.
- Define acceptance criteria.
- Identify system/data dependencies.
- Add Cobo safety boundaries for assets, policy, signing, custody, compliance, and agent autonomy.
- Add lane evidence fields.

## Do Not

- Overbuild.
- Turn every idea into a full system.
- Ignore seller/measurer dependencies.
- Treat AI as a feature button without redesigning workflow.
